# Story 43.2 — Web App Manifest

> **Epic 43: Progressive Web App (PWA)** | **Phase 6: Polish & Export** (Weeks 11-12)

## Description

As a player, I need to install the app on my device's home screen with a proper icon and app name. This story covers creating the web app manifest with correct metadata, generating app icons in all required sizes, adding iOS-specific meta tags, creating splash screens, and implementing a custom install prompt.

## Technical Context

- **App**: D&D Character Forge — local-first React PWA for D&D 5e character creation and management
- **Tech Stack**: React 18+, TypeScript, Vite, Tailwind CSS, shadcn/ui, Zustand (state), Dexie.js (IndexedDB), React Router, jsPDF (PDF export), Playwright (E2E testing)
- **Architecture**: No backend, pure client-side, offline-capable PWA, IndexedDB for persistence
- **Prior Phases Available**: Phases 1-5 (complete character creation, sheet display, session play, DM/campaign features)
- **Performance Targets**: Bundle <500KB, FCP <1.5s, TTI <3s, Lighthouse >90
- **Accessibility Target**: WCAG 2.1 AA compliance
- **Manifest Configuration**:
  - `name`: "D&D Character Forge"
  - `short_name`: "Char Forge"
  - `description`: "Create, manage, and play D&D 5e characters"
  - `start_url`: "/"
  - `display`: "standalone"
  - `background_color`: "#1a1a2e"
  - `theme_color`: "#e8b430"
  - `orientation`: "any"
  - `categories`: ["games", "entertainment", "utilities"]
- **Icon Sizes**: 192x192 (Android home), 512x512 (Android splash), 180x180 (Apple touch), 32x32 and 16x16 (favicons), maskable variant, SVG favicon
- **iOS Support**: Apple doesn't fully support web app manifest — needs meta tags in HTML
- **Install Prompt**: Custom banner using `beforeinstallprompt` event (Chrome/Edge), dismissable, not shown again after dismissal

## Tasks

- [ ] **T43.2.1** — Create `manifest.webmanifest` (auto-generated by vite-plugin-pwa) with:
  ```json
  {
    "name": "D&D Character Forge",
    "short_name": "Char Forge",
    "description": "Create, manage, and play D&D 5e characters",
    "start_url": "/",
    "display": "standalone",
    "background_color": "#1a1a2e",
    "theme_color": "#e8b430",
    "orientation": "any",
    "categories": ["games", "entertainment", "utilities"]
  }
  ```
- [ ] **T43.2.2** — **App icons:** generate icon set from a single source image (d20 die or the app logo):
  - 192x192 PNG (Android home screen)
  - 512x512 PNG (Android splash screen)
  - 180x180 PNG (Apple touch icon)
  - 32x32 and 16x16 favicon
  - Maskable icon variant (safe zone for adaptive icons on Android)
  - SVG favicon for modern browsers
- [ ] **T43.2.3** — **iOS-specific meta tags:** Apple doesn't fully support the web app manifest. Add:
  ```html
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  <meta name="apple-mobile-web-app-title" content="Char Forge">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">
  ```
- [ ] **T43.2.4** — **Splash screens:** generate Apple splash screen images for common iOS device sizes (iPhone, iPad). Or use the `theme_color` + icon fallback which is simpler and covers most cases
- [ ] **T43.2.5** — **Install prompt:** create an `InstallPWA` component that detects the `beforeinstallprompt` event (Chrome/Edge) and shows a custom install banner: "Install D&D Character Forge for offline access!" with "Install" and "Not now" buttons. Don't show on subsequent visits if dismissed

## Acceptance Criteria

- Web app manifest is generated with correct name, short_name, display mode, colors, and categories
- App installs as "D&D Character Forge" on Android and "Char Forge" on iOS
- Standalone display mode removes browser chrome when installed
- Background color (#1a1a2e) and theme color (#e8b430) are applied correctly
- All required icon sizes are generated: 192x192, 512x512, 180x180, 32x32, 16x16, maskable, SVG
- Maskable icon has proper safe zone for adaptive icons on Android
- iOS meta tags are present in the HTML head for Apple web app support
- Apple status bar style is "black-translucent" (gold-tinted status bar matching theme)
- Custom install prompt appears on Chrome/Edge for first-time visitors
- Install prompt has "Install" and "Not now" buttons
- Dismissed install prompt does not reappear on subsequent visits
- App is installable on Chrome, Edge, and iOS Safari

## Identified Gaps

- **Mobile/Responsive**: iOS Safari doesn't support beforeinstallprompt — no custom install prompt possible, only manual "Add to Home Screen"
- **Edge Cases**: Behavior when manifest generation by vite-plugin-pwa fails not specified
- **Edge Cases**: Apple splash screen generation for all iOS device sizes is complex and may not cover all devices
- **Accessibility**: Install prompt banner accessibility (focus management, ARIA) not specified

## Dependencies

- Story 43.1 (service worker must be configured for the app to be installable)

## Notes

- vite-plugin-pwa can auto-generate the manifest from configuration in `vite.config.ts`
- The maskable icon should have at least 10% safe zone padding around the important content
- iOS Safari doesn't support `beforeinstallprompt` — users must manually "Add to Home Screen" from the share sheet
- The install prompt dismissal should be stored in IndexedDB preferences to persist across sessions
- The icon source image should be a d20 die or app logo that looks good at all sizes (16px to 512px)
